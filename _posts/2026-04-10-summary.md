# Summary
In learned robotic control, there are two different classes of algorithms. There are model-based algorithms that rely on models of the dynamics of the robot to make decisions on what actions would be most optimal. The second type of algorithm is model-free, which is an algorithm that does not rely on a dynamics model to make a decision on the next action. 

Each of these different styles of these algorithms has its advantages and disadvantages. Model-based control is generally more computationally expensive, but they also tend to be more sample efficient. They also inherently have more of an ability to plan, and they might also have the ability to be more generalizable if the dynamics are modeled well. 

Model-free robotic control, as discussed earlier, learns a policy that picks actions that optimize some return on reward over time. This allows the model not to rely on an explicit dynamics model. This, of course, also has its own advantages and disadvantages. One of the main advantages is that it is relatively simple to implement, it is robust to model errors, and it has strong asymptotic performance. Having high asymptotic performance means that, given enough data, these types of models tend to exceed or match the performance of model-based controllers. The downside of these models is that they tend to be more sample inefficient, they don't implement any planning, which makes them myopic to long-term actions, and they tend to have bad generalization. 

## Problem Setup

### Problems specific to the paper 
The problem that is focused on in both of these papers is how we can leverage the positive aspects of both model-based and model-free learned control algorithms. In the first paper by Hansen and others, they introduce a method that they believe uses the advantages of both these frameworks, and in the second paper, they expand on their algorithm to make it even more performant. 

### General robotic control problem setup
In this paper the consider the classic robotic control setup, which is as follows. Consider the infinite-horizon Markov Decision Process (MDP) characterized by the tuple <script type="math/tex">(\mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R},\gamma,p_{0})</script>, where <script type="math/tex">\mathcal{S}\in \mathbb{R}^n</script> and <script type="math/tex">\mathcal{A}\in\mathbb{R}^m</script> are continuous state and action spaces. <script type="math/tex">\mathcal{T}: \mathcal{S} \times \mathcal{A}\times\mathcal{S} \to \mathbb{R}_{+}</script> is the transition function between an observation and the next observation given some action. <script type="math/tex">\mathcal{R}: \mathcal{S}\times\mathcal{A} \to \mathbb{R}</script> is a reward function, <script type="math/tex">\gamma \in[0,1)</script> is a discount factor, and <script type="math/tex">p_{0}</script> is some initial state distribution. The model from the paper aims to learn a parameterized mapping <script type="math/tex">\Pi_{\theta}: \mathcal{S} \to \mathcal{A}</script> with parameters <script type="math/tex">\theta</script> such that the discounted return is maximized along a trajectory.

## Method
In this section, I will try to describe the two papers and the methods that they take to attempt to take advantage of the positives of both model-based and model-free learned robotic control algorithms. 

### TD-MPC
In this paper, Hansen and others introduce Temporal Difference learning for Model Predictive Control, which, they claim, allows the model to take advantage of the benefits from both model-based and model-free algorithms. This algorithm is a framework for data-based model predictive control (MPC) dynamics model and a value function that are both learned using temporal differencing learning. This allows the algorithm to bootstrap the planning by using the learned single-step reward function to estimate the short-term reward, and then the value function to learn the longer-term rewards given some action. 

#### Key Technical Contributions 
Other models attempt to do something similar to this model, but there are several key contributions. Instead of predicting states or visual representations, they have the robot learn the dynamics in some latent space, which they believe to be more efficient. To do this, they include an encoder component that transforms observations from state space to some higher-dimensional observation space. They also back propagate through multiple rollouts, which helps to get rid of some of the compounding error that other world model style algorithms suffer from. 

#### Technical Details 
It is first helpful to go over Q learning on a more intuitive level. The idea is that we try to learn some function Q that can estimate the reward that might be received over an infinite horizon if the most optimal action is taken at each step. This function is parameterized by <script type="math/tex">\theta</script> and is an estimate of the true value function. We generally learn an approximation to this function through updating <script type="math/tex">\theta</script> to minimize the difference between the value the actual reward using a basic l2 norm. Of course there is also a discount thrown in there, but this is the idea of the Q network learning that is done in the TD-MPC paper as well as many others.

Model Predictive Control is a method to find locally optimal trajectories by finding an action that maximizes the reward for some time horizon.

TD-MPC effectively combines these things by learning a world model in latent space as well as a value model, which are then used iteratively by the model predictive path integral controller to find an optimal trajectory. 

There are basically 5 learned components of TD-MPC:
- **Representation:** <script type="math/tex">z_{t}=h_{\theta}(s_{t})</script>
- **Latent Dynamics**: <script type="math/tex">z_{t+1}=d_{\theta}(z_{t},a_{t})</script>
- **Reward**: <script type="math/tex">\hat{r}_{t}=R_{\theta}(z_{t},a_{t})</script>
- **Value**: <script type="math/tex">\hat{q}_{t}=Q_{\theta}(z_{t}, a_{t})</script>
- **Policy:** <script type="math/tex">\hat{a}_{t}\sim \pi_{\theta}(z_{t})</script>

With the loss functions:

<script type="math/tex; mode=display">
\begin{align}
\mathcal{L}(\theta; \mathcal{T}_i) 
&= c_1 \left\| R_\theta(\mathbf{z}_i, \mathbf{a}_i) - r_i \right\|_2^2 \\
&\quad + c_2 \left\| Q_\theta(\mathbf{z}_i, \mathbf{a}_i) 
- \left( r_i + \gamma Q_{\theta^-}(\mathbf{z}_{i+1}, \pi_\theta(\mathbf{z}_{i+1})) \right) \right\|_2^2 \\
&\quad + c_3 \left\| d_\theta(\mathbf{z}_i, \mathbf{a}_i) - h_\theta(\mathbf{s}_{i+1}) \right\|_2^2
\end{align}
</script>

Where the first term is the reward loss the second term is from the value loss and the third term is the dynamics loss. 

## Results
They found that on many of the task benchmarked their model was able to perform the same or better. Their model was also able to perform signifigantly better on some tasks that are more complex such as Humanoid, Dog, and Bin Picking. 

They also show that their model benefits well from data sharing, meaning that it is able to use its knowledge from one task and apply it to another one in a similar domain.
